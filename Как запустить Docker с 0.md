**1.Установить Git**
sudo apt install git -y
**Проверка:**
git --version

**2.Настроить Git**
git config --global user.name "Твое имя"  
git config --global user.email "твоя почта"
**Проверка:**
git config --list

**3.Установка Docker**
sudo apt install docker.io -y

**4.Запуск:**
sudo systemctl start docker  
sudo systemctl enable docker
**Проверка:**
docker run hello-world
Если выводит сообщение типа: "permission denied while trying to connect to the Docker daemon socket" - значит не хватает прав доступа, надо добавить **sudo** перед командой

Docker работает через сокет: /var/run/docker.sock
👉 Доступ к нему есть только у:
 -root
-пользователей в группе `docker`

**daemon** — фоновый сервис
👉 Docker = клиент + демон
- ты вводишь `docker run`
- клиент обращается к демону
- демон запускает контейнер
# Правильное решение
Добавь себя в группу docker:
**sudo usermod -aG docker $USER**
- `usermod` — изменить пользователя
- `-aG` — добавить в группу
- `docker` — имя группы
- `$USER` — текущий пользователь
**Затем необходимо перезапустить систему и снова ввести docker run hello-world

**Как проверить, что Docker реально запущен**
sudo systemctl status docker

**Когда убедились, что докер работает, можно поднять nginx контейнер и открыть его в браузере
## Поднять веб-сервер в контейнере

docker run -d -p 8080:80 nginx 
## `docker`
👉 Это CLI (Command Line Interface)
**CLI (Command Line Interface)** — интерфейс командной строки  
👉 программа, через которую ты управляешь Docker

---
## ▶️ `run`
👉 команда запуска контейнера
📌 Что делает:
- ищет образ
- если нет → скачивает
- создаёт контейнер
- запускает его

---
## ⚙️ `-d`
👉 сокращение от **detached**
**detached mode** — режим “в фоне”

---
### 📌 Что это значит:
- контейнер работает в фоне
- терминал освобождается
---
### ❗ Без `-d`:

docker run nginx

👉 ты “залипнешь” в процессе nginx

---

## 🌐 `-p`

👉 сокращение от **port**

---

## 📌 `8080:80`

Это самая важная часть 👇

---

### 📥 Формат:

<порт_на_твоем_ПК>:<порт_внутри_контейнера>

---

### 👉 В твоем случае:

8080:80

---

### 📌 Значит:

|Где|Порт|
|---|---|
|Твой компьютер|8080|
|Контейнер|80|

---

### 🧠 Как это работает:

браузер → localhost:8080 → Docker → контейнер → nginx:80

---

## 🌍 Почему 80?

👉 Потому что:

- **HTTP (HyperText Transfer Protocol)** работает на порту 80
- nginx внутри контейнера слушает 80

---

## 📦 `nginx`

👉 это имя образа

---

📌 Образ = шаблон контейнера

---

## 📌 Что делает Docker:

1. ищет образ nginx локально
2. если нет → скачивает с Docker Hub
3. создаёт контейнер
4. запускает nginx

---

# 🧠 Вся команда в переводе

Запусти контейнер nginx в фоне и пробрось порт 8080 на порт 80 контейнера

---

# 🔥 Что реально произошло

Ты сделал:

1. скачал nginx
2. запустил сервер
3. открыл доступ через порт
4. сделал мини-деплой
